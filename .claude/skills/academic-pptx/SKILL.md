# Academic Presentations Skill – Summary

This skill governs **content and structure** for academic presentations (conference talks, thesis defenses, seminars, grant briefings). It works alongside the PPTX technical skill.

## Core Workflow

1. **Identify type**: Default to "Structured Argument" for academic contexts
2. **Plan before building**: Create a slide outline; run the "ghost deck test" (action titles alone should tell the full argument)
3. **Apply design standards**: White backgrounds, one sans-serif font, minimal color palette, left-aligned text, consistent grid
4. **Build and QA**: Follow technical PPTX procedures plus academic-specific checks (action titles present, citations included, conclusions as final slide, etc.)

## Key Rules

- **Action titles, not topic labels** — every slide title states a complete takeaway
- **One insight per slide** — one exhibit per results slide with direct annotation
- **Minimal text** — ~40 words max body text; slides support speech, not replace it
- **Cite everything** — in-slide citations for borrowed figures; References slide required
- **End on Conclusions** — never finish on "Thank You" or blank slides

## Design Standards

| Aspect | Standard |
|--------|----------|
| Background | White |
| Font | One sans-serif (Arial, Calibri, Helvetica) |
| Colors | Dark navy primary, mid-blue accent, white background |
| Body text | 20 pt, left-aligned |
| Avoid | Decorative icons, clip art, full-bleed images on content slides |
